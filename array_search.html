Array Searching Operations


What is Searching?

Searching refers to find a particular element in the array if it is present in that array or not.

Types of searching:-

Here you'll learn about two searching algorithms-

  1. Linear search
  2. Binary search

Linear Search Algorithm

Linear search is one of the simplest searching method. In this process, we'll check each element of the array one-by-one until we find the element or reach the end of the array.

The alogorithm for linear search is given below-

FUNCTION linearSearch(A, n, item)
    FOR i FROM 0 TO n-1 DO 
        IF A[i]==item THEN
            RETURN i 
    ENDIF
    ENDFOR
    RETURN -1
END FUNCTION
  1. The function takes A, a linear array, n is the size of the array and item is the element to be searched here
  2. A for loop iterates through all the indices from 0 to n-1
  3. An if statement is used to check for each element of the array if the element at index i is equal to the item
  4. If it found a match, it returns the index i
  5. If the loop completes without finding the element. it will return -1 that denotes item is not found

C 

#include<stdio.h>

int linearSearch(int A[], int n, int item) {
    for (int i = 0; i < n; i++) {
        if (A[i] == item) {
            return i; // Returns the index if item is found
            }
        }
    return -1; // Return -1 if the item is not found
}
                            
int main() {
    int A[] = {10, 17, 28, 94, 121, 404};
    int n = sizeof(A) / sizeof(A[0]);
    int item;
                            
    printf("Enter the element you want to search: ");
    scanf("%d", &item);
                            
    int res = linearSearch(A, n, item);
                            
    if (res == -1) {
        printf("Element not found in the array.\n");
    } else {
            printf("Element found at index: %d\n", res");
    }
                            
    return 0;
    }

Advantages of Linear search

  • Simplicity:- Linear search is one of the simplest searching algorithm to understand as well as implement.
  • Avoid sorting:- Linear search doesn't required sorting of the elements before searching.

Disadvantages of Linear search

  • Time complexity:- The time complexity of LSA(Linear Search Algorithm) for average and worst case is O(n), n is the number of elements in the array.
  • Not suitable for large amount of data:- As per the array size gets increased, the performance of LSA(Linear Search Algorithm) is decreases significantly.

Time & Space Complexity

  1. Best case:- O(1)
  2. Worst case:- O(n)
  3. Average case:- O(n)
  • Space complexity:- O(1), since no additional space is required.

Binary Search Algorithm

Binary Search is a highly efficient algorithm for finding an item from a sorted list of items. It works by repeatedly dividing the portion of the list that could contain the item in half until you've narrowed the possible locations to just one. This method is significantly faster than linear search, especially for large datasets.

The alogorithm for BInary search is given below-

FUNCTION linearSearch(A, n, item)
    FOR i FROM 0 TO n-1 DO 
        IF A[i]==item THEN
            RETURN i 
    ENDIF
    ENDFOR
    RETURN -1
END FUNCTION
  1. The function takes A, a linear array, n is the size of the array and item is the element to be searched here
  2. A for loop iterates through all the indices from 0 to n-1
  3. An if statement is used to check for each element of the array if the element at index i is equal to the item
  4. If it found a match, it returns the index i
  5. If the loop completes without finding the element. it will return -1 that denotes item is not found

C 

#include<stdio.h>

int binarySearch(int A[], int n, int item) {
    int start = 0;
    int end = n - 1;
    while (start <= end) {
    int mid = start + (end - start) / 2; // To avoid overflow
    if (A[mid] == item) {
    return mid; // Return the index of the target element
    }
    else if (A[mid] < item) {
    start = mid + 1; // Search in the right half
    }
    else {
    end = mid - 1; // Search in the left half
    }
    }
    return -1; // Return -1 if the target element is not found
    }

Advantages of Binary search

  • Lower time complexity:- Binary search has a time complexity of O(log n) that make it more efficient searching algorithm than linear search.
  • Less no. of comparisons:- This algorithm eliminates half of the remaining elements in each steps to find the element by reducing number comparisons.

Disadvantages of Binary search

  • Need sorted data items:- Binary search is efficient for sorted data only. If the data items are not sorted, we have to sort the list of data first that may lead to O(n log n) of time.
  • Complex to implement:- Implementing binary search is much complex how ever the concept is simple and it also may be difficult to understand to new programmers.

Time & Space Complexity

  1. Best case:- O(1)
  2. Worst case:- O(log n)
  3. Average case:- O(log n)

Space complexity

  1. O(1):- When we use iterations.
  2. O(log n):- when we use recursion.
