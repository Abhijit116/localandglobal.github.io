Chapter 3:Introduction to Stack


What is Stack?

Stack is a linear data structure that operates on the Last In, First Out (LIFO) principle.
Stack allows only one data item to access at a time that is the topmost data item.

Example:- We can visual a stack of books as an example in which if we want to pick any book, we have to start removing from the top one.

Operations on Stack

  1. PUSH:- Used to add an element to the top of the stack.
  2. POP:- Used to remove an element from the top of the stack & also returns it.
  3. Overflow:- It is an error that occurs when we try to add elements in a stack that is already full.
  4. Underflow:- It is an erro that occurs when we try to remove elements from a stack that is already empty.

Implementing Single Stack using array:-

-: Steps :-

  1. Set top to the value -1
  2. For PUSH operation:
    • If the stack is not full, increase the value of top and PUSH the value to the top of the stack.
    • If the stack is full then show the overflow message.
  3. For POP operation:
    • If the stack is not empty return the top most element and decrement the value of top.
    • If the stack is empty then show the underflow message.



for (int i = 0; i < 5; i++)
{
printf("%s ",fruits[i]);
}

Applications of Stack:-

  1. Memory management:- Stacks help track the memory allocation and deallocation for dynamic memory allocation.
  2. Evaluating expressions:- Stacks are used to evaluate particularly postfix(Reverse Polish Notation) expression.
  3. Managing function call:- Stacks are used to control function calls and maintaining order of execution.
  4. Undo mechanism:- Text editors and other applications employ stacks to implement the undo functionality.

Limitations of array representation of stack:-

  1. Fixed Size:- While using array to implement stack, the stack has to be defined at compile time, that may lead to Overflow situation.
  2. Contiguous Memory Allocation:- Array takes Contiguous memory blocks that may not always be available.This may lead to allocation failure.
  3. Inefficient Operations:- If shifting is needed for insertion or deletion of elements, that might be inefficient, since for stack operations i.e. POP & PUSH time complexity is O(1).
  4. Non-dynamic:- Array is a non-dynamic data structure. But if we need to manage dynamic data using stack(implemented using array) can be error prone as compared to stack implementation using linked list.
